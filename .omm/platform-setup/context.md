Each platform driver implements a `Platform` interface (`src/lib/platforms/types.ts`) with four methods: `detect()` checks if the tool is installed (usually via `which <binary>` or checking for config directories); `isSetup()` checks if the current version is already registered; `setup()` writes the skill manifest/plugin files to the project or global config dir; `teardown()` removes those files. Version-matching in `isSetup()` ensures that `omm update` followed by `omm setup` will re-register with the new version. The skills source is resolved at runtime by `utils.ts` which searches for the `skills/` directory relative to the package install location. Claude Code registers via `.claude-plugin/`, Cursor via `.cursor-plugin/`, others use their own conventions. The `marketplace.json` in `.claude-plugin/` enables the Claude Code plugin marketplace listing.
