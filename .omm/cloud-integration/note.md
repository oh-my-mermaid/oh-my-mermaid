The two-level org model (user default org in credentials + per-project org in config.yaml) allows users to belong to multiple orgs while keeping per-project cloud routing simple. The `OMM_API_URL` override was added primarily for integration testing against a local dev server. The credentials file is written with mode 0o600 explicitly to prevent other OS users from reading the token — this is a Unix-only protection; Windows does not honor the mode flag.
