Authentication uses a Bearer token stored in `~/.omm/credentials.json` (mode 0o600). The token is obtained via `omm login` which prompts the user to authenticate through the browser (or similar flow) and then saves the returned token. The API base URL defaults to `https://ohmymermaid.com` but can be overridden via `OMM_API_URL` environment variable (useful for self-hosted or staging instances). Push serializes the entire `.omm/` directory by walking it recursively and sending all files as `{ path, content }` pairs in a single JSON POST. Pull is the reverse — receives the file list and writes them all locally. The org/project binding uses two levels: a per-user default org in credentials and a per-project slug in `.omm/config.yaml`. Git commit hash is attached to push payloads for traceability.
