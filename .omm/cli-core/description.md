The CLI entry point and core data layer for oh-my-mermaid. Handles command dispatch from the `omm` binary, reads/writes class fields to the `.omm/` directory, manages the config and per-class metadata (meta.yaml), and provides the store API used by all other subsystems. Implements lazy-init (auto-creates .omm/ on first write) and enforces the 7-field schema (description, diagram, constraint, concern, context, todo, note).
