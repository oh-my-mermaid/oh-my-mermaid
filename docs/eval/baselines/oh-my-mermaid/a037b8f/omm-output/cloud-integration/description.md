Manages authentication and synchronization with the omm.dev cloud service. Handles login (token-based auth stored in `~/.omm/credentials.json`), push (upload entire `.omm/` directory to cloud), pull (download and overwrite local `.omm/`), link (associate project with a cloud slug), share (print public URL), and org management (list orgs, switch default org, view members). Credentials are stored per-user in the home directory; the project-cloud binding is stored in `.omm/config.yaml`.
