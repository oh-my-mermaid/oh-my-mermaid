The diff engine (`src/lib/diff.ts`) implements a lightweight regex-based Mermaid parser (`parseMermaid`) that extracts node IDs and edge pairs without a full AST. It handles `graph`/`flowchart` declarations, skips `classDef`, `click`, `style`, `subgraph`/`end`, and matches common arrow syntaxes (`-->`, `--o`, `--x`, `---`, `-.->`, `==>`, `~~>`). The previous diagram is stored in `meta.yaml` (`prev_diagram` field) when a new diagram is written. The refs engine (`src/lib/refs.ts`) scans for `@word` patterns using a regex; it supports both forward lookup (what does this class reference?) and reverse lookup (what references this class?). The validate engine (`src/lib/validate.ts`) enforces the omm diagram convention: `graph LR` declaration, balanced brackets (quote-aware), edge labels required, known `classDef` names (`external`, `concern`, `entry`, `store`) with canonical colors, no self-refs, no dangling refs, and 5-15 node count recommendation.
