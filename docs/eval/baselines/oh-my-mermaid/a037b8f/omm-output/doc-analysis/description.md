The document intelligence layer: parses, diffs, cross-references, and validates Mermaid diagrams stored in `.omm/`. Provides three capabilities: (1) diff — compares current diagram against the previously saved version in meta.yaml and reports added/removed nodes and edges; (2) refs — scans diagram text for `@class-name` patterns to build an inter-class reference graph (incoming and outgoing); (3) validate — checks diagrams against a rule set covering graph declaration, bracket balance, edge labels, classDef names/colors, self-references, dangling refs, and node count. Used by both the CLI commands (`omm diff`, `omm refs`, `omm validate`) and the local viewer API.
