The CLI is the single binary entry point (`omm`). All commands funnel through `src/cli.ts` which does raw `process.argv` parsing with a switch statement — no commander/yargs dependency. The store (`src/lib/store.ts`) is the persistence layer: it manages `.omm/<class>/` directories with fixed filenames per field type (`diagram.mmd`, `description.md`, etc.). Meta tracking is handled by `src/lib/meta.ts` which captures git commit/branch at write time. The store uses lazy-init: read commands return an error if `.omm/` is absent; write commands auto-create it. YAML (via the `yaml` package) is used for `config.yaml` and `meta.yaml`. All other subsystems (viewer, cloud, platforms, doc-analysis) depend on the store API.
