The store's lazy-init design (write creates, read errors) is intentional: it prevents `omm list` or `omm status` from silently creating empty `.omm/` directories in projects that haven't been scanned yet. The single `yaml` dependency was chosen deliberately to keep the install footprint minimal. The `GLOBAL_COMMANDS` array in `cli.ts` acts as a guard: anything not in that list with 2+ args is treated as a class-field write, enabling the `omm <class> <field>` protocol used by AI skills.
